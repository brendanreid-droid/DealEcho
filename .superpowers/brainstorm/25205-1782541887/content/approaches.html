Three approaches to Enterprise billing + team model

$13/seat/month, minimum 5 seats. Multiple managers. Email invites.

A ★

Stripe quantity subscription + Firestore teams collection

Recommended. One Stripe price at $13/seat with quantity = seat count (min 5). Stripe handles proration natively when quantity changes. Firestore gets a dedicated teams collection — members, roles, invites all live there. Clean, scalable, minimal custom billing logic.

Pros

  • Stripe manages all proration math
  • Clean data model
  • Seat changes = one Stripe API call

Cons

  • New Stripe price object needed
  • New Firestore collection + security rules
B

Two Stripe items (flat $65 base + $13 addon per extra seat)

Subscription has two line items: a fixed $65/mo base + a $13/seat item for seats above 5. Preserves the "5 included" mental model explicitly in Stripe.

Pros

  • Invoices clearly show "base + extras"
  • Easy to change base price independently

Cons

  • More complex Stripe subscription structure
  • Two items to manage on every change
  • No real benefit over approach A at this pricing
C

Fixed subscription + manual invoice items for overages

Enterprise starts as a flat $65/mo subscription. When seats exceed 5, create one-off Stripe invoice items at billing cycle end for the overage.

Pros

  • Simplest initial Stripe setup

Cons

  • Must build custom metering/overage tracking
  • Delayed billing = revenue leakage risk
  • Complex webhook handling
